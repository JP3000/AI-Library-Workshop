# Iterative Chain Prompt Framework（ICPF）
## 循环式链式提示框架：科研研究设计与论文写作工作流
> 基于 Prompt Engineering 与 Chain Prompting 构建的**可迭代、可验证、可追踪**科研写作流程框架
> 灵感来源：Jay Alammar & Maarten Grootendorst《Hands-On Large Language Models — Chapter 6: Prompt Engineering》

## 一、项目定位
ICPF 是面向研究者、学生及科研写作者的结构化提示词系统，核心目标是**引导使用者自主完成科研全流程构建**，而非直接生成论文内容。框架覆盖从研究问题萌芽、方法选型，到论文撰写、质量校验的完整链路，适配人文社科、理工科等多学科科研写作场景。

## 二、核心价值
1. **结构化降门槛**：将复杂的科研写作拆解为标准化步骤，降低科研入门与流程管理成本；
2. **循证性保严谨**：每一步骤均包含「操作依据、检索式、引用占位」，确保研究过程可验证；
3. **迭代循环驱动**：核心思路是**「初步想法 → AI 提供检索建议 → 文献检索与阅读 → 细化输入 → 下一步」**。文献阅读是连接各步骤的实质桥梁，而非简单的线性生成；
4. **全流程覆盖**：从研究起点确认到论文定稿校验，覆盖科研写作全生命周期。

## 四、使用流程
### 前置准备
仅需明确两个核心输入：
- **Topic**：研究主题（如「乡村振兴背景下农村电商的发展路径」）
- **Context**：研究场域与对象（如「某省县域农村电商从业者、政府相关部门」）

### 完整执行步骤（Pre-Step → Step 10）
1. **Pre-Step**：生成理论框架、研究方法、逻辑链、伦理考量、质量保障方案；
2. **Step 0**：全局设置，确认研究背景、核心术语定义；
3. **Step 1**：明确研究起点，梳理已有研究基础与空白；
4. **Step 2**：确认研究语境与核心概念，统一概念界定；
5. **Step 3**：构建核心研究问题（RQ），拆解子问题；
6. **Step 4**：比较不同研究方法适配性，确定最终方法；
7. **Step 5**：设计研究方案（提供A/B/C多套备选）；
8. **Step 6**：设计数据/资料收集工具（如访谈提纲、问卷、编码表）；
9. **Step 7**：规划资料分析思路，确定分析方法与流程；
10. **Step 8**：设计结论章节结构，明确结论呈现逻辑；
11. **Step 9**：完成文献综述（从提纲到定稿）；
12. **Step 10**：全文整合与写作计划制定，质量自检与修订。

## 五、使用原则
1. **循证优先**：所有研究设计与写作内容需基于可验证的事实、文献，拒绝无依据假设；
2. **迭代优化**：每一步输出均可作为下一步的输入，也可基于新的思考回溯修订前序步骤；
3. **工具定位**：框架仅为科研写作的辅助工具，最终研究结论与论文质量由使用者的专业判断决定；
4. **灵活适配**：可根据研究类型（量化/质化/混合研究）、学科特点，调整步骤侧重点与模板内容。
