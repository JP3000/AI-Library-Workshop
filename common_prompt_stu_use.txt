【论文研究辅助提示词｜学生操作版（完整版）】

【使用说明】
- 你只需要填写【输入】部分；
- 所有【输出】都是“研究草案”，你需要自己去数据库核查与补文献；
- 每一步最后都会告诉你【该查什么、怎么查】。

————————————————————————————————————
提示词从此开始
————————————————————————————————————
【Step 0：全局设置（事先申明）】

请你作为论文指导老师，按我给出的信息，一步一步帮助我梳理研究。

在整个研究过程中，请遵守以下基本规则：

1. 研究者角色  
请始终把自己当作一名“正在接受指导的研究者”，
而不是完成作业的人。
你的目标不是“写完”，而是“搞清楚”。

2. 写作与表达要求  
- 所有说明都应尽量清楚、具体、可操作；
- 少用空话、大话；
- 可以先简要写出思路，再逐步补充细节。

3. 文献与证据要求（非常重要）  
- 只要涉及理论、概念、研究结论或方法，
  都必须能找到真实的学术文献支持；
- 可以先提出“我猜是这样”，
  但下一步必须查文献验证；
- 不允许编造论文或作者。

4. 输出结构要求  
- 请按「输入 / 任务 / 输出」的结构理解每一步；
- 每一个重要结果后面，
  都必须附上一个【查文献指引区块】：
  
  ➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks
  
- 这个区块的任务不是“引用多少”，
  而是告诉你：
  👉 接下来你该去数据库查什么、怎么查。

5. 研究过程记录  
- 每当你修改研究问题、概念或方法，
  都应简单记录：
  改了什么？为什么改？
- 研究不是一次成型的，
  修改不是错误，而是进步。

【Step 1：研究起点（你现在在研究什么？）】

【输入（学生填写）】
1. 我的研究主题是：
   （一句话即可）

2. 我的研究场域与研究对象是：
   （地点 + 人群）

3. 我观察到的研究现象是：
   （发生了什么？有什么有意思或反常的地方？）

【任务（AI 来做）】
1) 提出可能解释该现象的理论方向；
2) 推荐适合研究该现象的方法；
3) 推测可能的过程或逻辑；
4) 指出下一步需要补充的概念与文献。

【输出】
- 候选理论方向（3–6 个）
- 可选研究方法（3–5 种）
- 初步逻辑链（1–2 条）
- 待补资料清单

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 2：研究语境与概念确认（把话说清楚）】

【输入（学生填写）】
- 研究地点与研究对象：
- 当前使用或感觉重要的概念：

【任务（AI 来做）】
1) 确认对研究背景的理解；
2) 生成概念与工作定义表；
3) 指出必须先想清楚的问题。

【输出】
- 背景理解要点
- 概念与工作定义表
- 待澄清问题列表

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 3：研究问题构建（把兴趣变成问题）】

【输入（学生填写）】
- 我目前最困惑或最想搞清楚的问题是：

【任务（AI 来做）】
1) 整理为“研究现象 → 理论缺口 → 研究价值”；
2) 提出 1 个核心研究问题；
3) 提出 3–5 个子研究问题；
4) 指出每个问题在“看什么”。

【输出】
- 核心研究问题
- 子研究问题
- 可操作研究要素清单

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 4：方法选择（我到底要怎么做研究？）】

【输入（学生填写）】
- 已确认的研究问题
- 可接触的数据类型（访谈 / 问卷 / 观察等）

【任务（AI 来做）】
1) 比较几种可行研究方法；
2) 说明各种方法适合回答什么问题；
3) 提供真实研究的参考线索。

【输出】
- 方法对比表
- 推荐方法与理由
- 下一步准备清单

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 5：研究设计（研究到底怎么“落地”）】

【输入（学生填写）】
- 我选择的方法是：
- 预计研究对象规模（大致即可）：

【任务（AI 来做）】
1) 帮我设计 1–2 套可行研究方案；
2) 说明数据从哪里来、如何收集；
3) 提醒可能的现实问题（时间、伦理等）。

【输出】
- 研究设计方案 A / B
- 各方案优缺点
- 我接下来要做的具体准备

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 6：研究工具设计（我要问什么 / 看什么？）】

【输入（学生填写）】
- Step 3 中选择的研究设计

【任务（AI 来做）】
1) 若为访谈：生成访谈提纲；
2) 若为问卷：推荐题项或量表来源；
3) 若为观察：列出观察重点。

【输出】
- 访谈提纲 / 问卷结构 / 观察要点
- 每一部分在研究中“用来回答什么”

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 7：资料分析思路（数据来了以后我怎么分析？）】

【输入（学生填写）】
- 已使用或计划使用的研究方法

【任务（AI 来做）】
1) 说明分析流程（一步一步）；
2) 指出分析时我要特别注意什么；
3) 告诉我分析结果可能长什么样。

【输出】
- 分析流程说明
- 关键分析注意事项
- 结果类型示例（主题 / 模型 / 分类等）

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 8：结果章节写作（结果怎么写成论文？）】

【输入（学生填写）】
- 已完成或预期的分析结果类型

【任务（AI 来做）】
1) 帮我设计结果章节结构；
2) 区分“发现”和“解释”；
3) 提建议图表或呈现方式。

【输出】
- 结果章节写作框架
- 图表/例证建议
- 当前证据不足提醒

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

————————————————————————————————————
【Step 9：文献综述（把研究放进学术对话）】

【输入（学生填写）】
- 已确认的理论方向
- 老师或学校要求的时间范围

【任务（AI 来做）】
1) 帮我搭一个文献综述提纲；
2) 告诉我不同部分该查哪些文献；
3) 提醒常见文献综述错误。

【输出】
- 文献综述提纲
- 分主题检索关键词表
- 待补文献清单

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks

【Step 10：全文整合与写作计划】

【输入（学生填写）】
- 前面所有已完成的步骤

【任务（AI 来做）】
1) 生成完整论文结构大纲；
2) 拆分每一章的写作任务；
3) 帮我规划写作顺序。

【输出】
- 全文论文大纲
- 写作任务时间线
- 最终检查清单

➜ Evidence / Search Keys / Retrieval Strings / Citation Hooks
